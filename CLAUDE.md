## Project Overview

**OdoTrack** — An offline-first, neo-brutalist PWA vehicle expense logger, fuel mileage calculator, live trip tracker, and maintenance manager with receipt scanning OCR.

## Tech Stack

- **React 19** + **TypeScript** + **Vite 6**
- **Tailwind CSS v4** with `@theme` / `@variant` — not the classic `tailwind.config.js` approach
- **IndexedDB** for local data persistence (no backend)
- **Tesseract.js** + **OpenCV.js** for client-side receipt OCR

## Build & Development

```bash
# Install dependencies
npm install

# Run dev server (port 3000, accessible on network)
npm run dev

# Production build
npm run build

# Preview production build
npm run preview

# Type-check (no emit)
npm run lint
```

## Architecture

### Data Layer (`src/db.ts`)

A single IndexedDB database (`OdoTrackDB`, version 1) with object stores: `vehicles`, `fuel_logs`, `trips`, `expenses`, `receipts`, `settings`. The `dbAPI` object exposes typed CRUD methods. Key behaviors:

- **Cascade delete**: Deleting a vehicle cascades to all related fuel logs, trips, and expenses.
- **Odometer sync**: Saving a fuel log, trip, or expense with an odometer value automatically updates the vehicle's current odometer.
- **Mileage recalculation**: Fuel logs are auto-sorted by odometer; `mileageSinceLast` is recalculated and stored after any insert or delete via `recalculateMileage()`.

### State Management (`src/App.tsx`)

All data lives in `App` state and is passed down as props. The `reloadAllData()` function re-fetches all stores and updates state — child components call this on every mutation via callbacks like `onLogAdded`, `onTripDeleted`, etc. Tab selection (`activeTab`) and selected vehicle (`selectedVehicleId`) are persisted to `localStorage`.

### OCR Pipeline (`src/ocrEngine.ts`)

Receipt scanning works entirely client-side:

1. **Image preprocessing**: OpenCV.js crops receipt contours, resizes, and applies bilateral filtering + adaptive thresholding. Falls back to a Canvas-based adaptive threshold (Bradley-Roth) if OpenCV fails or times out.
2. **Tesseract.js** recognizes text from the preprocessed image.
3. **Regex extraction** (`parseReceiptText`) extracts cost, litres, price/litre, date, odometer, and station name with INR-specific heuristics and mathematical self-consistency checks.

Both Tesseract and OpenCV are loaded on-demand from CDN, not bundled.

### UI Design System (`src/index.css`)

Neo-brutalist style: thick black borders (`border-4 border-black`), solid offset shadows (`.neo-shadow`, `.neo-shadow-lg`, `.neo-shadow-sm`), and bold uppercase typography (Space Grotesk). Dark mode toggles via a `.dark` class on `<html>`. Colors are defined in the `@theme` block using Tailwind v4's CSS-first config.

### Service Worker

The service worker is generated by `vite-plugin-pwa` using Workbox under the hood. It precaches all build assets and uses a cache-first strategy for CDN resources. On update, a banner prompts the user to reload.

The registration is handled in `src/serviceWorkerRegistration.ts` using `virtual:pwa-register`. `updateServiceWorker(true)` triggers the new SW activation and reloads the page.

### Tab-Based Navigation

Six tabs: **Dashboard**, **Fuel**, **Trips**, **Expenses**, **Vehicles (Garage)**, **Backup/Settings**. Each is a top-level component in `src/components/`. The bottom nav is sticky on mobile and floats on desktop.

## Important File Patterns

- **No `tailwind.config.js`** — styling is entirely CSS-based with `@theme` and `@variant` in `src/index.css`.
- **Component naming**: `NeoModal.tsx` and `ConfirmModal.tsx` are reusable neo-brutalist UI primitives.
- **Date format**: All dates in data are `YYYY-MM-DD` strings.
- **Currency default**: `INR` (set in `dbAPI.getSettings()` defaults and `App.tsx` initial state).
- **Sample seeder**: `dbAPI.seedSampleData()` populates the database with demo vehicles, fuel logs, trips, and expenses for testing.
